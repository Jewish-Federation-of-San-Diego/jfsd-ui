🔴 Cross-Cutting Issues (All 10 Dashboards)

Generic Titles

  • Every chart uses label titles ("Survival Curves", "Pipeline Funnel") instead of insight headlines
  • Fix: "FY25→FY26 retention averages 42% — pre-2015 cohorts retain 2x better" instead of "Survival Curves"
  • This is the single highest-impact change across all 10

Missing Data Freshness

  • DataFreshness component exists but most dashboards show asOfDate from JSON — if JSON is stale, user doesn't know
  • Fix: Show both data date AND render date: "Data as of Mar 5 · Rendered Mar 7"

Color Inconsistency

  • Uses PLOTLY_COLORS[0..6] array (decorative) instead of semantic colors
  • Teal should mean favorable, orange should mean unfavorable — but colors are assigned by array index
  • Fix: Map semantic meaning to color, not position

No Annotations

  • Zero call-outs, reference lines, or annotations on any chart
  • Decision matrix requires: max 2 per chart, always annotate the insight
  • Fix: Add Plotly annotations to key data points (largest value, anomaly, benchmark)

📋 Per-Dashboard Assessment

1. Major Gifts Pipeline Needs Work
  • Fix Pie chart for stage distribution — Decision matrix explicitly bans pie charts. Replace with horizontal bar sorted by value. This is the only chart crime across all 10 dashboards.
  • Improve Funnel chart for pipeline stages — Funnel implies sequential drop-off (which is correct for pipeline), but the funnel visualization in Plotly is often misleading with width proportions. Consider a horizontal bar with stage-based ordering instead.
  • Add Pipeline ratio context — Show 3:1 pipeline-to-goal benchmark line. The matrix prescribes: "Bullet chart (actual vs goal)" for pipeline metrics.
  • Improve KPI titles are generic ("Pipeline Value", "Close Rate"). Should state insight: "Pipeline at $2.8M needs 3x velocity to hit $9M goal".
Prescribed pattern: KPI-First (Executive). Funnel → horizontal bar. Pie → sorted horizontal bar. Add close-rate trend sparkline.
2. Cohort Survival Analysis Mostly Good
  • OK Line chart for survival curves — Correct for change-over-time with multiple cohorts.
  • OK Heatmap for retention — Good for showing density/pattern across cohorts × periods.
  • Improve All survival lines same color — PLOTLY_COLORS[0] used for every cohort. They'll overlap and be indistinguishable. Use distinct colors or small multiples.
  • Add Annotation on best/worst cohort — Call out which cohort retains best and which is hemorrhaging.
  • Improve Tabs ("Native Plotly" vs embedded) — confusing for end users. Pick one, remove the toggle.
Prescribed pattern: Analytical. Survival = change-over-time (line, correct). Consider small multiples if >5 cohorts to reduce clutter.
3. Donor Lifecycle Mostly Good

  • Improve Migration heatmap colorscale uses navy→blue. Should use diverging: teal for growth cells, orange for loss cells.
Prescribed pattern: Analytical. Lifecycle = flow. Consider Sankey for the migration instead of heatmap — more intuitive for non-analysts.
